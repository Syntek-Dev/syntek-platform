## Overview

This file documents the Prettier code formatter configuration for the project. Prettier automatically formats JavaScript, TypeScript, JSON, YAML, Markdown, HTML, and CSS files to maintain consistent code style across the team. Configure Prettier rules in `.prettierrc` and apply via CLI or pre-commit hooks.

---

## Installation

```bash
npm install
```

## Commands

```bash
npm run format          # Format all files
npm run format:check    # Check formatting (no changes)
npm run format:staged   # Format only staged files
```

## What Gets Formatted?

- ✅ JSON files (`.json`, `.jsonc`)
- ✅ YAML files (`.yml`, `.yaml`)
- ✅ Markdown (`.md`)
- ✅ HTML templates (`.html` - Django/Jinja)
- ✅ CSS files (`.css`)
- ✅ JavaScript (`.js`)
- ✅ GraphQL (`.graphql`)
- ❌ Python files (use Black instead)

## Auto-formatting

- **On save:** VS Code auto-formats (if extension installed)
- **On commit:** Pre-commit hook auto-formats staged files

## Config Files

| File                      | Purpose              |
| ------------------------- | -------------------- |
| `.prettierrc`             | Configuration        |
| `.prettierignore`         | Excluded files       |
| `package.json`            | Scripts & deps       |
| `.pre-commit-config.yaml` | Git hook integration |
